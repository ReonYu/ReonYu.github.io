简述

API网关是一个服务器，是系统的唯一入口。从面向对象设计的角度看，它与外观模式类似。API网关封装了系统内部架构，为每个客户端提供一个定制的API。它可能还具有其它职责，如身份验证、监控、负载均衡、缓存、请求分片与管理、静态响应处理。
API网关方式的核心要点是，所有的客户端和消费端都通过统一的网关接入微服务，在网关层处理所有的非业务功能。通常，网关也是提供REST/HTTP的访问API。服务端通过API-GW注册和管理服务。

  1. 方便客户端维护– 每个请求方不用管理多个api url，统一访问api-gateway即可

  2. 接口重构时调用方不须了解接口本身等拆分和聚合

  3. 客户端无须关心接口协议

  4. 统一权限控制、接口请求访问日志统计

  5. 安全，是保护内部服务而设计的一道屏障

  6. 开源-最大好处

wso2/carbon-apimgt

WSO2 API Manager使用经过验证的WSO2产品（例如WSO2 Enterprise Integrator 6.4.0，WSO2 Identity Server 5.7.0和Windows 2000）通过API创建，发布，生命周期管理，版本控制，货币化，治理，安全性等一系列功能克服了这些挑战。WSO2治理注册表。此外，它还由WSO2流处理器4.3.0提供支持，可立即用于大规模可扩展部署。

WSO2 API Manager是高性能的，完全开源的，并根据Apache软件许可版本2.0（当今最易用的商业许可之一）发布。它为开发团队提供了Web界面，以供开发团队部署和监视API，以及让消费者通过用户友好的店面订阅，发现和使用API​​。API管理器还提供完整的API治理，并与WSO2治理注册表共享相同的元数据存储库。如果您的设置需要管理的不仅仅是API，我们建议您使用WSO2 API管理器进行API治理，并使用WSO2治理注册表进行其他工件。密钥管理器的默认通信协议是Thrift。

WSO2 API管理器是一个正在进行的项目，每个新版本都不断进行改进和增强，以解决新的业务挑战和客户期望。WSO2邀请用户，开发人员和爱好者通过在线论坛，邮件列表和支持选项来参与或获得我们在许多不同级别的开发团队的帮助。

关键概念

  1. API管理器组件
    一个组件由一个或多个OSGi捆绑包组成。捆绑软件是OSGi中的模块化单元，类似于Java中的JAR文件。所有WSO2产品的基于组件的体系结构使开发人员可以灵活地删除或添加具有最小依赖性的功能。

  1. API发布者

API开发通常由了解API，接口，文档，版本等技术方面的人员完成，而API管理通常由了解API业务方面的人员进行。在大多数业务环境中，API开发是与API发布和管理不同的职责。

WSO2 API Manager提供了一个简单的Web界面，称为 WSO2 API Publisher， 用于API开发和管理。它是一种结构化的GUI，旨在供API创建者开发，记录，扩展和版本化API，同时还促进了与API管理相关的更多任务，例如发布API，货币化，统计分析和推广。
