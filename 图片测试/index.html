Flume数据采集异常：java.nio.charset.MalformedInputException: Input length = 1

  • Solution :

flume导入数据文件报错，原因是inputCharset属性的默认值UTF-8但是所读取的文件的字符集却不是UTF-8，可以通过设置source的属性值或者修改文件的字符集解决。

解决：

首先vi中查看文件字符集（注意：查看文件的编码不要用file -I 命令）

:set encoding/fileencoding

方法一：修改flume配置文件，增加source的输入字符集

agent.sources.r1.inputCharset = latin1

（本次使用方法一）

方法二：

vi中修改文件字符集

:set encoding/fileencoding=UTF-8

接口机数据延迟 : 以10:01开启的小时任务为例，处理的是9:00-10:00期间，flume监控目录导入kafka中的所有数据，但是处理完插入插入后，发现这期间监控目录中的数据（end_date时间）很大一部分是8:00-9:00（而非9:00-10:00）的数据，这部分最新数据因为延迟下发，在9:00-10:00期间进入监控目录，且延迟下发时间不确定

  • Solution :

接口机下发数据延迟无法解决，SparkStreaming实时流无法兼顾ProcessingTime和EventTime,仅能保证，系统时间9:00-10:00flume传入kafka的数据，能够被及时处理（数据本身的EventTime，并不做考虑）

Oracle数据导入异常

  • Solution :

描述：
使用yarn集群模式（进行测试）时，在客户端导入ojdbc连接包(注意要与oracle版本对应)，并且在spark-defaults.conf配置文件中添加路径（spark.executor.extraClassPath, spark.driver.extraClassPath），程序虽然不报错，但是经常只能添加部分数据
原因：
即便使用yarn模式，需要在将ojdbc连接包添加到NodeManager所在的所有节点的（spark-defaults.conf）配置路径下,否则当前只有有jar包的NodeManager可以连接oracle
解决方式：ojdbc连接包添加到NodeManager所在的所有节点的（spark-defaults.conf）配置路径下

异常重启后的数据处理策略

  • Solution :

程序中计算流程如下：

整点开启小时任务

① kafka小时数据à

② hive插入增量数据à

③ oracle结果表插入数据à

④ oracle监控表插入数据à

⑤ 计算结束，等待下一小时任务启动

方案一：offset存储在③④之间

方案二：offset存储在②③之间

最终决定将offset存储放在②③之间，存储的是offsetRange.untilOffset

原因如下：
方案一：最初考虑此方案是因为，最终目的是④oracle结果表插入数据，但是如果oracle异常，重启后，因为尚未saveOffset，所以这小时数据会加入下一小时数据重跑，因为Hive中已经有了该数据，会造成hive中当天数据重复，后面每小时经过sql脚本（直接处理hive）插入oracle的数据就都不正确了.

而且即便使用方案一，oracle出错时，那一小时的oracle处理结果也不可能在下一小时的处理中表示出来（根据业务逻辑，只能表示为下一小时的处理结果。）

方案二：最终选择此方案是因为，只要hive不出错，即小时数据已经插入hive里面了，重启时就不重跑这部分数据。虽然oracle出错，这一小时数据是没有的，但是因为hive中数据是存在的，可以手动调用脚本处理后，插入到oracle中。能够保证后面的每次计算都是正确的。

这样的好处是，在hive插入小时数据前的异常，可以在重启后重新加载并处理这部分数据，在hive插入小时数据后的异常（包括下一小时任务启动前的任何异停止），重启后不再重复消费这部分数据。

Hive表中的前一天数据未处理

  • Solution :

虽然00:00过后前一天的hvie表被清理，但是事实表明，00:00后还是会像监控目录导入前一天的数据。这些数据的文件名仍为当天（实际证明确是如此），但在当天处理范围内。

解决方法是hive表中增加了数据本身的end_date属性，在sql脚本处理过程中，对hive表中“当天”数据进行过滤

手动更改zookeeper中offset的程序

  • Solution :

需要写zkClient的连接程序，为保证hive表中当天数据完整性和唯一性，不建议修改次offset值，因为程序中的saveOffset是添加在小时数据插入hive之后的，按照程序已经插入zookeeper的offset值，无论任何时间程序停止，任何时间重启，都能保证hive表中当天数据的正确性。

Yarn模式的print输出位置，异常并不打印到控制台

  • Solution :

异常无法观察，可以使用local模式，将异常打出
